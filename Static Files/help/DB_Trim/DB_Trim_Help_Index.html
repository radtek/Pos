Database Trim.

 

The purpose of this application is to irretrievably remove information from the MenuMate Database archives, thus reducing the overall database size.

 You will lose all history that occurred before this date, including receipts, and the ability to report on this data. Therefore, you should create a new copy of the database that Office can use to access the historical data.

 The database path is retrieved from the MenuMate registry, so you will need MenuMate to have been installed in order for it to put this registry setting in place for you. When DBTrim is run, the database to be trimmed is shown at the top of the screen.

Here are the steps you need to complete in order to reduce the size of your database.

Before you begin, make a note of the oldest historical date that you wish to keep data for.

·         Back up the current database before you begin.  Use IBTools for this (double click on the MenuMate.IB or MenuMate.GDB file).

·         Rename the backup (Menumate.GBK) to append it with today's date as it will have all relevant information up to today's date eg. Menumate History 2007-05-30.GBK. Restore the backed up database to a .IB file.

·         Set up a company in office and point its MenuMate database path to your newly restored database.

·         Run a few reports and test this database is working for you.

·         Run DBTrim.exe and, using the Year / Month / Day picker, enter in your historical date.

·         Select [Trim DB] and read the warning.

·         Read it again. You will lose all sales archives that occurred before this historical date, receipts and everything.

·         If you are happy to proceed click [OK].

·         The application will let you know when it has finished.

·         Quit DBTrim, and back up and restore the trimmed database.

·         The Database’s size will not change until you complete this back up and restore to clean out the deleted records.
