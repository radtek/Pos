CollectForms method
Initiates subclassing for all child controls of the given Form, which are handled by the theme manager.

Applies to
TThemeManager

Declaration
procedure CollectForms(Form: TCustomForm = nil);

Description
This method is usually called by the theme manager itself but can also be used by the application if necessary (e.g. for forms in DLLs). The code takes care not to add a control twice and uses CollectControls to do the actual collection. If Form is assigned then only child controls of the given form are iterated, otherwise all forms in the application are searched.

Note
Since there can be more than one theme manager in the application it is important to ensure that only one does the actual window subclassing. This manager is the socalled main manager and only for this one instance is subclassing enabled. Hence if you call CollectForms (or CollectControls) of others than the main manager instance then nothing will happen. If the main manager is freed then another theme manager instance (if there is one) will automatically take over the main manager part.
